# Инструкции агента: создание лендинга

Полный процесс создания продающего лендинга — на основе работы над landing.html и gallup.html для Talents Navigator, обогащённый методологиями Linear, Superhuman, Notion, CopyHackers, Bob Moesta JTBD, StoryBrand и CXL.

---

## Шаг 0. VOC Research — голос клиента

**Самый важный шаг, который пропускают 90% команд.** Весь копирайтинг строится не на придуманных фразах, а на реальных словах пользователей.

**Принцип Joanna Wiebe (CopyHackers):** "Most copy is swiped from VOC data paired with swiped formulas." Хорошая копи — не писательство, а редактирование клиентского языка.

**Источники данных:**
- Отзывы на G2, Capterra, Trustpilot — на свои продукты и на конкурентов
- Reddit, Telegram-чаты, форумы по нише — неотфильтрованный язык
- Customer support транскрипты
- Email-ответы на onboarding-последовательность
- NPS follow-up комментарии
- Интервью с 8-12 реальными пользователями (Switch Interview, см. ниже)

**Ключевой вопрос для интервью (не спрашивать мнения — реконструировать историю):**
> "Расскажи как ты последний раз искал решение этой проблемы. Что происходило в твоей жизни, когда ты начал искать что-то подобное?"

**Четыре колонки для сбора данных:**

| Боль | Желание | Возражение | Ценности |
|---|---|---|---|
| Что расстраивает | К чему стремятся | Почему не купят | Что важно |
| → в Problem section | → в Solution | → в FAQ | → в tone of voice |

**Результат:** 30-50 живых фраз пользователей, разложенных по четырём колонкам.

## Шаг 1. JTBD-анализ продукта

До написания копирайтинга — изучить сам продукт и формализовать, какие работы он выполняет.

**Что делать:**
- Выписать все функции продукта (табы, механики, форматы вывода)
- Для каждой функции задать вопрос: в какой ситуации пользователь это использует и зачем?
- Написать JTBD-сценарий по формату:

```
РАБОТА: [короткое название работы]
Контекст: [ситуация, в которой возникает работа]
Триггер: [конкретный момент, когда человек понимает что нужен инструмент]
Функциональная работа: [что именно нужно сделать]
Эмоциональная работа: [как человек хочет себя чувствовать]
Социальная работа: [как хочет выглядеть в глазах других]
Как продукт закрывает: [конкретные фичи которые решают эту работу]
Барьеры к использованию: [что мешает]
```

**Результат:** 5–8 JTBD-сценариев, покрывающих основные кейсы использования.

## Шаг 2. Four Forces — анализ переключения

Для каждого JTBD-сценария заполнить модель четырёх сил (Bob Moesta). Это определяет не только что написать, но и где на лендинге это написать.

```
JTBD: [название работы]

PUSH (что толкает от текущего решения):
→ Боль которую нельзя игнорировать
→ Момент "хватит терпеть"
→ Пример для Talents Navigator: "Читаю отчёт третий раз — всё равно не понимаю что делать"

PULL (что тянет к новому решению):
→ Желаемая картина жизни после
→ Конкретный результат который хочется
→ Пример: "Хочу войти в разговор с руководителем с конкретными данными о себе"

ANXIETY (что тормозит переключение):
→ Страхи и сомнения: "а вдруг...", "а что если..."
→ Издержки переключения
→ Пример: "А вдруг это ещё один тест личности без практического смысла"

HABIT (что держит в статус-кво):
→ Почему текущее решение всё ещё терпимо
→ Инерция и привычки
→ Пример: "Могу просто ещё раз перечитать отчёт Gallup"
```

**Как это конвертируется в структуру лендинга:**

| Сила | Секция лендинга | Что писать |
|---|---|---|
| Push | Problem section | Боль, frustration, "знакомая ситуация?" |
| Pull | Solution + Features | Картина жизни после, конкретные результаты |
| Anxiety | FAQ + Proof | Работа с возражениями, social proof |
| Habit | CTA | Снижение commitment: "займёт 5 минут", "бесплатно" |

**Переключение происходит когда Push + Pull > Anxiety + Habit.** Если лендинг не конвертирует — скорее всего слабо проработана одна из этих сил.

## Шаг 3. Positioning Contrast — против чего позиционируемся

Сильное позиционирование всегда имеет видимый контраст. Linear — против Jira (bloat). Superhuman — против бесплатного Gmail. Notion — против "один инструмент на всё не работает".

**Определить до написания текста:**

1. **Против какого статус-кво?** (не конкурент, а текущее поведение пользователя)
   - Пример: Talents Navigator vs "PDF-отчёт который лежит в папке"

2. **Против какого конкурента** (если есть прямой)?
   - Что конкурент делает плохо, что мы делаем лучше — один конкретный контраст, не список

3. **Против какого убеждения?**
   - Пример: "Галлап — это тест на личность без практического применения" — именно это убеждение лендинг должен разрушить

**Записать одной строкой:** "Talents Navigator — это [что], а не [от чего уходим]."

Этот контраст должен читаться в hero h1 или h2, не быть закопанным в features.

## Шаг 4. Профили ЦА на основе JTBD

Из сценариев вывести архетипических пользователей.

**Что делать:**
- Сгруппировать похожие JTBD-сценарии (у одного пользователя может быть несколько работ)
- Для каждой группы описать профиль: имя, роль, контекст, основная боль, что нанимает продукт делать
- Определить: кто первичная аудитория (получает больше всего ценности), кто вторичная
- Для каждого профиля заполнить Four Forces (Шаг 2)

**Результат:** 5 профилей с описанием основной работы, боли, триггера входа и четырёх сил.

---

## Шаг 5. Messaging Hierarchy

До написания текста — определить иерархию сообщений. Что первое, что второе, что третье.

**Шаблон:**

```
Главное сообщение (hero h1):
→ Самый сильный push или самый желанный pull из Four Forces
→ Проверка: это про клиента или про нас?

Второе сообщение (hero sub или первая секция):
→ Конкретизация: что именно получит + за какое время

Третье сообщение (solution/features):
→ Как именно это работает — конкретные механики, не абстракции

Контрсообщение (FAQ/proof):
→ Главное возражение (anxiety) и как мы с ним работаем
```

**StoryBrand check:** В hero — клиент герой, продукт — проводник. Проверить: первое слово в заголовке — "вы/ты" или название продукта? Если второе — переписать.

## Шаг 6. Первая версия текста

На основе самого сильного JTBD-сценария написать лендинг.

**Что делать:**
- Взять язык из VOC Research (Шаг 0) — не придумывать, а редактировать реальные фразы
- Опираться на messaging hierarchy (Шаг 5)
- Выбрать 1 сценарий с наибольшим эмоциональным зарядом и широким охватом аудитории
- Написать полную версию лендинга (все секции, RU + EN)
- Угол: конкретная боль из выбранного сценария → продукт как решение

**Specificity check:** Заменить все "streamline", "boost", "powerful", "seamless" на конкретику. Плохо: "ускорьте работу". Хорошо: "50ms interactions", "ответ за 5 минут вместо 30". Конкретность = доверие.

---

## Шаг 7. Вторая версия текста

Альтернативный угол входа — другой JTBD или другая эмоциональная ставка.

**Критическ�� важно:** версия 2 — это другая дверь в тот же дом, не другой дом.

- Hero, problem, messaging hierarchy меняются — это угол входа
- Features, solution, proof **покрывают весь продукт** — только акценты смещаются
- Нельзя убирать целые части продукта ради нарратива версии 2

**Частая ошибка:** взять JTBD одной фичи (например "настроить AI") и построить вокруг неё всю страницу, убрав остальные фичи. Результат — промо отдельной фичи, не продукта. Если такая страница получается — это отдельная feature-page, не версия лендинга.

**Что делать:**
- Взять второй по силе JTBD-сценарий или перефреймировать главный
- Написать альтернативную версию hero + problem секций (другая боль, другой триггер)
- В features: поменять порядок карточек и акценты в описаниях под новый угол — но все фичи остаются
- В proof: выбрать отзывы, резонирующие с новым углом
- Зафиксировать: чем версия 2 отличается по углу входа, а не по составу продукта

## Шаг 8. 5-second test

**До коридорных тестов** — проверить первое впечатление. 55% посетителей проводят на странице меньше 15 секунд.

**Протокол:**
1. Показать hero-секцию 3 разным людям на 5 секунд
2. Убрать, спросить: "Что делает этот продукт? Для кого он? Что запомнилось?"
3. Если ответ нечёткий — переписать header до коридорных тестов

**Инструменты:** fivesecondtest.com, Maze, Lyssna — или просто показать на экране вживую.

**Проверка Clarity (LIFT Model, Peep Laja):** "Clarity trumps persuasion." Если за 5 секунд непонятно что предлагается — никакая техника убеждения не поможет.

---

## Шаг 9. Коридорные тесты

Прогнать обе версии через профили ЦА — только после того как версии написаны и прошли 5-second test.

**Что делать:**
- Для каждого из 5 профилей пройти по версии 1, затем по версии 2
- Записать для каждого профиля:
  - Где спотыкается (непонятный термин, нелогичный переход, непонятный CTA)
  - Какой ключевой вопрос остаётся без ответа (незакрытая anxiety из Four Forces)
  - Что конвертирует (какой элемент убеждает нажать CTA)
- Сравнить: какая версия работает лучше для первичной аудитории

**Результат:** таблица «профиль × версия → находки».

**Wynter-принцип:** 12-13 ответов = насыщение. Больше не нужно — количество инсайтов не растёт. Качественное тестирование говорит *почему* что-то не работает (в отличие от A/B теста, который говорит *что* работает лучше).

---

## Шаг 10. Финальная версия

Взять лучшее из двух версий по итогам тестов.

**Что делать:**
- Определить базовую версию (та, что лучше сработала на первичную аудиторию)
- Внести точечные правки из тестов: заменить спотыкающиеся элементы, добавить ответы на незакрытые вопросы
- Не смешивать углы двух версий — выбрать один основной, второй использовать точечно
- Проверить по LIFT Model: Value Proposition, Relevance, Clarity, Urgency, Anxiety, Distraction

## Пример: JTBD-сценарии Talents Navigator

---

**РАБОТА: Перевести Gallup-отчёт в рабочие инсайты**
Контекст: Человек только что получил результаты CliftonStrengths — список из 34 тем с описаниями. Описания дженерические, написаны маркетинговым языком, не понятно что с ними делать в реальной работе.
Триггер: Читает отчёт третий раз и понимает, что дальше абстрактных описаний не продвигается. "Ладно, у меня Strategic — и что?"
Функциональная работа: Понять, что именно эти таланты означают лично для него — не в абстракции, а в конкретных рабочих ситуациях. Разобраться, какие таланты давать волю (z1-2), какие осознанно применять (z3), какие не трогать (z4-5).
Эмоциональная работа: Перейти от ощущения "я потратил деньги на тест, но так ничего и не понял" к "теперь я знаю, на что опираться".
Социальная работа: Быть человеком, который осмысленно говорит о своих сильных сторонах, а не просто называет цифры из отчёта.
Как продукт закрывает: Лидерский профиль с персонализированными инсайтами по топ-10 талантам (доверие, забота, опора, направление). Зональная система z1-z5 — визуально и однозначно показывает, что природное, а что нет. Цитаты.
Барьеры к использованию: Нужно сначала купить и пройти Gallup-тест (~$20-25). Инвайт-система — не всегда есть инвайт.

**Four Forces:**
Push: "Читаю отчёт и понимаю, что дальше его не продвигаюсь"
Pull: "Хочу знать конкретно — что мне делать с этим профилем в работе"
Anxiety: "Боюсь что это ещё один тест без практического применения"
Habit: "Могу просто ещё раз перечитать Gallup-отчёт сам"

---

**РАБОТА: Настроить AI под свой стиль мышления**
Контекст: Активный пользователь ChatGPT или Claude, использует их ежедневно. Замечает, что ответы часто "не попадают" — слишком общие, не в том формате, не в том ритме.
Триггер: После очередной переписки с AI где пришлось 5 раз уточнять "нет, объясни иначе" — понимает, что хочет один раз правильно настроить контекст.
Функциональная работа: Получить готовый системный промпт, который точно описывает его способ думать — и вставить в AI за 2 минуты.
Эмоциональная работа: Почувствовать, что AI работает как настроенный инструмент, а не как случайный собеседник.
Социальная работа: Быть человеком, который умеет работать с AI эффективно.
Как продукт закрывает: Таб "Инструкции для AI" — генерирует системный промпт (короткий и расширенный). Таб "AI-усиление" — промпты для каждого таланта.
Барьеры к использованию: Нужно доверять логике Gallup. Скептик к "тестам на личность" не увидит ценности до использования.

**Four Forces:**
Push: "Снова трачу 20 минут на переформулировки вместо того чтобы получить ответ сразу"
Pull: "AI отвечает так, как будто знает как я думаю — с первого раза"
Anxiety: "Это займёт много времени настраивать и всё равно не сработает"
Habit: "Привык уточнять — так работают все с AI"

---

**РАБОТА: Диагностировать причину выгорания**
Контекст: Специалист или руководитель, у которого работы много, но энергии нет. Формально всё нормально — хорошая компания, нормальные коллеги, достойная зарплата. Но ощущение что "не своё".
Триггер: После отпуска не восстановился. Или на встрече поймал себя на мысли "зачем я вообще этим занимаюсь".
Функциональная работа: Понять конкретно — что именно в нынешней среде работает против его природы. Получить проверяемую гипотезу, а не совет "найди баланс".
Эмоциональная работа: Перейти от тревожного "со мной что-то не так" к "ситуация понятна, есть конкретные причины".
Социальная работа: Иметь аргументированный разговор с руководителем — не "я устал", а "вот что конкретно не работает".
Как продукт закрывает: Таб "Чекап среды" — 10 вопросов, диагностирующих среду. Зональная система — показывает сколько времени уходит на z4-5 задачи.
Барьеры к использованию: Результат — диагноз, а не план. Нужно управлять ожиданиями.

**Four Forces:**
Push: "Уже третий месяц в таком состоянии — что-то явно не так"
Pull: "Хочу понять точно почему устаю, чтобы можно было это изменить"
Anxiety: "Вдруг окажется что проблема во мне, а не в среде"
Habit: "Можно потерпеть ещё немного и само пройдёт"

---

**РАБОТА: Подготовиться к разговору с руководителем о роли**
Контекст: Человек понимает, что текущая роль не раскрывает его. Хочет поговорить о расширении ответственности или изменении функции. Боится выглядеть "капризным".
Триггер: Приближается performance review. Или коллегу продвинули, а его — нет.
Функциональная работа: Сформулировать конкретный разговор: вот мои сильные стороны, вот где создаю ценность, вот что нужно от среды.
Эмоциональная работа: Войти в разговор уверенно, с опорой на данные — не как проситель, а как человек с позицией.
Социальная работа: Выглядеть зрелым профессионалом, который говорит про вклад, а не "хочу больше".
Как продукт закрывает: Лидерский профиль — секции "как выстраивать доверие", "как давать направление". Связки талантов — показывает уникальные комбинации.
Барьеры к использованию: Продукт даёт сырой материал, но не скрипт разговора.

**Four Forces:**
Push: "Performance review через неделю — надо что-то конкретное сказать о себе"
Pull: "Хочу войти в разговор с аргументами, а не просто попросить"
Anxiety: "Руководитель не знает Gallup — не поймёт о чём я"
Habit: "Поговорю в общих чертах — как обычно"

---

**РАБОТА: Понять как управлять конкретным человеком в команде**
Контекст: Руководитель видит, что человек способный, но что-то не работает — задачи не те, обратная связь не заходит, мотивация падает.
Триггер: После провала задачи, которую руководитель считал простой. Или при найме нового сотрудника.
Функциональная работа: Получить конкретные инструкции — как ставить задачи, как давать обратную связь, что не делать — для человека с конкретным профилем.
Эмоциональная работа: Чувствовать себя руководителем, который работает с людьми осмысленно.
Социальная работа: Выглядеть в глазах команды как тот, кто видит людей.
Как продукт закрывает: Лидерский профиль — секция "управление командой". Зональная система — помогает понять куда направлять человека.
Барьеры к использованию: Руководитель должен знать Gallup-профиль сотрудника.

**Four Forces:**
Push: "Уже второй раз сотрудник завалил задачу которую я считал очевидной"
Pull: "Хочу наконец понять как с ним работать — и перестать угадывать"
Anxiety: "Это займёт время — надо изучать чей-то профиль, потом применять"
Habit: "Дам задачу как обычно и посмотрю что выйдет"

---

**РАБОТА: Делегировать задачи из слабой зоны через AI**
Контекст: Человек понимает, что ряд задач даётся тяжело — детальное планирование, администрирование, холодные контакты. Делегировать некому.
Триггер: Неделю прокрастинировал задачу из z4-5. Или потратил два часа на то, что коллега делает за 20 минут.
Функциональная работа: Получить конкретные промпты для AI, которые позволяют закрыть типичные задачи из слабой зоны.
Эмоциональная работа: Перестать корить себя за слабые зоны. Принять что это не слабость, а неприродное.
Социальная работа: Оставаться эффективным, не раскрывая "слабые места" публично.
Как продукт закрывает: Таб "AI-усиление", раздел "компенсаторы" — готовые промпты под каждый талант из z4-5.
Барьеры к использованию: Нужно принять, что у тебя есть z4-5. Для перфекционистов — сдвиг мышления.

**Four Forces:**
Push: "Снова откладываю этот отчёт уже три дня — надоело себя заставлять"
Pull: "Если AI закроет мои слабые зоны — смогу сфокусироваться на том что реально умею"
Anxiety: "AI сделает плохо и мне всё равно придётся переделывать"
Habit: "Сделаю сам как умею — не идеально, но приемлемо"

## Шаг 11. Структура страницы

**Стандартная структура лендинга:**
1. Header (sticky, logo, nav, CTA, lang switcher)
2. Hero (eyebrow, h1, subtitle, CTA-кнопки, hero-note, hero-visual)
3. [Образовательная секция] — если продукт требует пояснения (что такое Gallup, что такое JTBD и т.д.)
4. Problem (тёмный блок, боль пользователя — Push из Four Forces)
5. Solution (ценностное предложение, 4 буллита + визуал — Pull)
6. Features (карточки возможностей, 5 штук)
7. How It Works (3 шага)
8. Social Proof (3 отзыва — закрывает Anxiety)
9. Final CTA (тёмный блок — снижает Habit)
10. Footer

**Принципы:**
- Каждая секция отвечает на один вопрос пользователя
- Problem всегда тёмный фон — визуально отделяет боль от решения
- Final CTA повторяет hero-посыл, но с другим углом (не повторять дословно)
- StoryBrand: клиент = герой, продукт = проводник (Йода, не Люк)

## Шаг 13. Копирайтинг — русская версия

**Принципы (выработаны с RU-редактором):**

- Никаких клише: "синергия", "рычаги роста", "инновационное решение", "рабочая система" — запрещено
- Никаких калек с английского: "чекап" → "диагностика", "AI-усиление" → конкретное описание пользы
- Не открывать с отрицания ("Вы не сломаны") — это защитная позиция, а не приглашение
- Hero h1: активное действие, не констатация. "Вы играете в чужую игру" сильнее чем "Вы в чужой игре"
- CTA: конкретное обещание, не команда. "Ваш профиль — за пять минут" лучше чем "Начните прямо сейчас"
- Invite-only: живая инструкция ("попросите коллегу") а не системное сообщение об ошибке
- "Система" не должна повторяться больше 2 раз на странице — размывается
- Specificity: "профиль с конкретными действиями за 5 минут" а не "рабочая система лидерства"

**Шаблон hero:**
- H1: сильный Push или желанный Pull из Four Forces (1-2 строки)
- Sub: что делает продукт + конкретный результат + временные рамки
- CTA primary: глагол + конкретный результат ("Получить профиль", "Посмотреть свой профиль")
- Hero-note: ограничения и контекст (бесплатно, что нужно, для кого)

## Шаг 14. Копирайтинг — английская версия (US market)

**Принципы (выработаны с EN-редактором):**

- "Powered by X" а не "Based on the X methodology"
- CTA: "See Your X →" или "Get Your X →" а не "Build Your X →" (пользователь не должен работать)
- Invite-only: "Early access · Invite only" — эксклюзивность, не барьер
- Не использовать guillemets («»): в EN-тексте имена собственные без кавычек
- "Talent synergy" → "How your themes work together" (синергия = buzzword 2012 года)
- "Concrete" как усилитель — калька с русского "конкретный", не используем
- "Playbook" вместо "concrete set of actions"
- "Play to your strengths" а не "operate through your strengths"
- Финальный CTA должен перекликаться с hero (тематическая арка)
- Социальное доказательство для скептика: цифры (50 лет исследований, Fortune 500, миллионы пользователей)
- Superhuman-принцип: позиционирование строится на желаемой идентичности, а не на функциях

## Шаг 15. Мультиязычность (i18n)

**Реализация через data-атрибуты:**

```html
<!-- В HTML: каждый текстовый узел оборачивается -->
<h2><span data-i18n="section_title">Текст на русском</span></h2>
```

```javascript
// В JS: один объект с переводами
const translations = {
  ru: { section_title: 'Текст на русском', ... },
  en: { section_title: 'English text', ... }
};

function switchLang(lang) {
  document.documentElement.lang = lang;
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.dataset.i18n;
    if (translations[lang]?.[key]) el.innerHTML = translations[lang][key];
  });
  document.getElementById('lang-btn').textContent = lang === 'ru' ? 'EN' : 'RU';
  document.title = lang === 'ru' ? 'RU Title' : 'EN Title';
  localStorage.setItem('lang', lang);
  window._currentLang = lang;
}

// Инит при загрузке
const savedLang = localStorage.getItem('lang') || 'ru';
window._currentLang = savedLang;
if (savedLang === 'en') switchLang('en');
```

**Кнопка переключения:**
```html
<button class="lang-switcher" id="lang-btn"
  onclick="switchLang(window._currentLang === 'ru' ? 'en' : 'ru')">EN</button>
```

**Что НЕ переводить:**
- Имена собственные (Gallup-темы из отчёта пользователя — берутся как есть)
- Технические термины которые одинаковы в обоих языках (Claude, n8n)

**Что переводить обязательно:**
- Все видимые текстовые строки включая заголовки страниц (`document.title`)
- `aria-label` атрибуты (доступность)
- Hero-visual с демо-данными (пример талантов должен быть на языке интерфейса)
