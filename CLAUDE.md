# Talents Navigator — правила разработки

## Обязательно перед каждым деплоем

**Запускать агента-тестировщика перед тем как показывать результат пользователю.**

Агент должен проверить:
1. JS синтаксис (`node --check`)
2. Что изменённая логика корректна — прочитать код и проверить все зависимости (переменные, ключи объектов, названия полей в БД)
3. Соответствие данных — если код обращается к JSON файлам или Supabase, проверить что ключи совпадают с реальными данными
4. Edge cases — пустые данные, null, несуществующие поля

**Никогда не говорить "готово, проверяй" без прохождения этих проверок.**

## Частые ошибки которые уже были

- Ключи доменов в коде не совпадали с русскими названиями в `t.json` (было `'Executing'`, нужно `'Исполнение'`)
- `isWeak` объявлен в одной функции, но не в другой (`renderFullCard` vs `renderSimpleCard`)
- URL параметр `?invite=` в admin.html, но код читал `?token=`
- RLS рекурсия: политика читала `profiles` внутри политики на `profiles` → infinite recursion
- Профиль администратора не существовал в БД → `is_admin` не читался

## Структура проекта

- `index.html` — основное приложение (single-file, vanilla JS)
- `admin.html` — админка (управление пользователями и инвайтами)
- `content/` — JSON файлы с данными (загружаются из Supabase Storage после авторизации)
- `test.js` — юнит-тесты (`npm test`)

## Данные

Домены в `t.json` на **русском**: `Исполнение`, `Влияние`, `Выстраивание отношений`, `Стратегическое мышление`

Supabase: `kyjwdwnooufskjsyhpts.supabase.co`
Таблицы: `profiles` (user_id, themes, is_admin), `invites` (token, note, used_at, used_by)
